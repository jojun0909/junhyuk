This project started as a school programming project. The goal was to create a room assignment program that could be used practically and also show the assigned room layout visually. I designed the assignment algorithm and implemented several key management features.

Period 2025.04.25 – 2025.06.10
Team 3 members
Role Room assignment algorithm, reward/penalty system, student room-change feature
  • Grouped students based on sleep-time and heating/cooling preferences.
  • Sorted students by reward/penalty score so that students with higher scores could receive more favorable assignments.
  • Handled remaining one or two students by moving them to the next group to avoid unfair room sizes.
  • Assigned rooms by considering each room's Manhattan distance from the supervisor's room.
  • Implemented reward/penalty adjustment and student room-change features.

Algorithm highlight: students were first divided into preference groups, then sorted by score conditions, grouped into rooms of two or three, and finally assigned to physical room positions based on fairness and distance from the supervisor's room.

Review: I learned how an actual program is planned, divided into features, implemented, and improved through a full project process.

Period 2025.11.03 – 2025.12.04
Team 2 members
Role Overall game structure, tile system, save/load feature
  • Implemented the overall game structure using a game class.
  • Managed the status of each land tile through a tile class.
  • Represented the map with nested lists and dictionaries.
  • Stored information such as ground state, ownership, treasure, crops, working status, clicks, and locked tiles.
